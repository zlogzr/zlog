大模型不"知道"答案，它在按概率补全"看起来最像答案"的文本。流畅和正确是两回事，所以幻觉不是偶发 bug，而是生成机制的固有产物——你的目标不是消灭它，而是把不确定性收敛到可控范围。

## 为什么会幻觉

模型优化的是"预测下一个 token"，不是"检索事实"，也没有"我不知道"这个默认动作。在训练分布里，自信流畅的回答比承认无知更像高质量文本，于是它会用合理的格式包裹错误内容：伪造的 API 签名、不存在的论文、看似精确实则编造的版本号。

规律很清楚：越是细节、长尾、需要精确引用的问题越危险；越是常识、有海量重复语料的问题越可靠。

## 哪些场景必须核实

| 场景 | 风险 | 处理 |
|---|---|---|
| 思路、改写、头脑风暴 | 低 | 直接用 |
| 代码逻辑、配置项 | 中 | 跑一遍 / 编译验证 |
| API 签名、参数、版本号 | 高 | 查官方文档 |
| 法律、医疗、财务、安全 | 极高 | 人工 + 权威源 |

> 答错代价高、又无法一眼判真伪的，一律默认它在幻觉。

## 怎么对抗

- **要来源和不确定度**：让它对每个事实标注依据，并显式说出哪些没把握。能力边界处的"不确定"信号，往往比答案本身更有价值。
- **可验证地提问**：把问题转成能被验证的形式——要可运行的代码、可查的链接、可复算的步骤，而不是一段结论散文。
- **接地（grounding）**：把权威材料喂进上下文，让它基于给定文本作答而非凭记忆。提示词示例：

```
基于我粘贴的官方文档回答。
1) 只用文档里的内容，不要补充你记得的；
2) 每个结论标出处段落；
3) 文档没写就回答"未提及"，不要猜。
```

- **交叉验证**：高风险结论换个问法重问，或用搜索/另一会话复核，多源一致才采信。
- **用工具兜底**：能编译、能运行、能检索的，就别拿对话当事实库——让外部工具来裁决真伪。
